2 Hyperlinks

You can link from one game to another game fairly easily. What you’ll need is:

1) A player entity with a (link-follower) component

2) A link entity (e.g. a door or a portal) with a (link "PATH") component. Where "PATH" should reference another folder that contains a main.rkt file.

3) The main.rkt file should provide a function called enter whose type is (-> game? (listof entity?)). This function will be dynamically required when the player touches the entity with the link component. The returned list of entities will will become the new game state. The old game is passed in and can be used to dynamically adjust the list of entities (e.g. for carrying the player entity into the new game).

A relatively simple example can be found in this repo:

https://github.com/thoughtstem/RacketCon2018
